# Көпағынды бағдарламалау (Java Multithreading)

## Жоба туралы мәлімет
* **Тақырыбы:** Thread және Runnable интерфейстерімен жұмыс істеу негіздері.
* **Мақсаты:** Бірнеше процесті бір уақытта параллель орындау (мысалы, файл жүктеу немесе есептеу).

---

## Теориялық сұрақтарға жауаптар
1. **Thread деген не?**
   * Thread (Ағын) — бұл бағдарламаның орындалуындағы ең кіші бөлік немесе «жеңіл процесс».
   * Бір бағдарлама (процесс) ішінде бірнеше ағын қатар жұмыс істей алады.
   * Әрбір бағдарламаның кем дегенде бір негізгі ағыны болады (Java-да ол main ағыны).

2. **Runnable интерфейсі не үшін қажет?**
   * Ол Java-дағы "көптік мұрагерлік" шектеуін айналып өтуге көмектеседі.
   * Егер класс басқа бір кластан мұрагерлік алған болса, сіз Thread класын кеңейте алмайсыз, сондықтан Runnable қолданылады.
   * Runnable тапсырманың логикасын (`run()` әдісі) сипаттайды, ал Thread сол тапсырманы іске қосу тетігі болып табылады.

3. **Көпағындылықтың артықшылықтары қандай?**
   * **Өнімділік:** Күрделі есептеулерді бірнеше бөлікке бөліп, процессордың бірнеше ядросында қатар орындау арқылы уақытты үнемдейді.
   * **Жауап беру жылдамдығы:** Фондық режимде ауыр тапсырмалар (мысалы, файл жүктеу) орындалып жатқанда пайдаланушы интерфейсі қатып қалмайды.
   * **Ресурстарды тиімді пайдалану:** Процессордың бос тұрып қалмауын қадағалайды.

---

## Ағынның өмірлік циклі
Ағын келесі кезеңдерден өтеді:
1. **New:** Ағын нысаны құрылды.
2. **Runnable:** `start()` әдісі шақырылды.
3. **Running:** `run()` әдісі орындалуда.
4. **Blocked:** Күту немесе блоктау күйі.
5. **Dead:** Жұмыс аяқталды.

---

## Практикалық бөлім (Код сипаттамасы)
Бағдарламада `Runnable` интерфейсін іске асыратын `MyTask` класы қолданылады. Негізгі бағдарлама екі бөлек ағынды іске қосады:
* **Ағын 1:** "Файл жүктеу" тапсырмасын орындайды.
* **Ағын 2:** "Есептеу жұмысы" тапсырмасын орындайды.

Графиктік салыстыру бойынша, ағындар саны артқан сайын тапсырманың орындалу уақыты 1000 мс-тан 320 мс-қа дейін қысқарады.
